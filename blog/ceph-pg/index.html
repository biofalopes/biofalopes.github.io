Bash Script to slowly increase pg_num in a Ceph pool

Bash script that gradually increases the pg_num/pgp_num of a pool.

September 28, 2021

2-Minute Read

LinkedIn

Email

Twitter

Reddit

I needed to increase the pg_num of our biggest pool in Ceph, used for object storage, and we needed to gradually increase in small steps to avoid putting too much stress on the cluster.

Our pool had 4096 PGs and we wanted to increase it to 8192. We don’t have pg autoscaler enabled, but even if we had it would set the value straight to 8192, which is not desired and also not recommended by Red Hat.

Increasing the number of PGs is the most intensive process that can be performed on a Ceph cluster, and can have drastic performance impact if not done slowly. Once the data starts moving for a chunk of PGs, it cannot be stopped or reversed.

The instructions of how to perform the operation can be found at https://access.redhat.com/solutions/2457321 .

Here’s how I approached the task:

#!/bin/bash

DATE='date +%d/%h/%Y-%H:%M:%S'
TARGET=8192
INCREMENT=2
INTERVAL=120m
POOL=target_pool

while true; do
  if [ "$(ceph -s | grep misplaced)" != "" ]
  then
    echo "$($DATE) There are misplaced objects. Waiting..."
    sleep 30m
  else
    echo "$($DATE) No misplaced objects. Increasing number of PGs."
    PG_NUM=$(ceph osd pool get $POOL pg_num | awk -F ': ' '{print $2}')
    if [ $PG_NUM -eq $TARGET ]
    then
      echo "$($DATE) Operation Completed. PG number is $PG_NUM."
      exit 0
    else
      echo "$($DATE) Current PG number is $PG_NUM."
      PG_NUM=$((PG_NUM+INCREMENT))
      echo "$($DATE) New PG number is $PG_NUM."
      ceph osd pool set $POOL $PG_NUM || { echo "$($DATE) Failed setting pg_num" ; exit 1; }
      echo "$($DATE) PG number increased. Waiting 2h until next iteration..."
      sleep $INTERVAL
    fi
  fi
done
  1. I start checking if there are misplaced objects in the cluster; if so, i wait 30 more minutes before checking again;
  2. With no misplaced objects, I check the current pg_num and then compare it with my target pg_num;
  3. If it’s still below the target, I increment the pg_num by $INCREMENT, 2 in my case, and run a ceph osd pool set $POOL $PG_NUM;
  4. After that, I wait for 2 more hours. When it reaches the target, the script exits with 0;
  5. If any command returns an error, the script exits;
  6. I write a log file to monitor the execution.

After the conclusion, we then need to increase the pgp_num in the same slow fashion, so we can tweak the script and change the parameter.
